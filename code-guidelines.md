# コード規約ガイドライン

## 基本方針
- 可読性を重視し、自己説明的なコードを書く
- コンポーネントと機能を適切に分離する
- 型安全性を確保する
- パフォーマンスに配慮する

## 命名規則

### 変数・関数
- キャメルケース（camelCase）を使用する
- 変数名は内容を的確に表現する具体的な名前を付ける
- 真偽値を返す変数・関数は `is`, `has`, `should` などのプレフィックスを使用する
- イベントハンドラー関数は `handle` または `on` で始める
  - 例: `handleSubmit`, `onSearchChange`

### コンポーネント
- パスカルケース（PascalCase）を使用する
- 名詞または名詞句を使用する
  - 例: `TranslationList`, `FileUploader`, `SearchBar`

### フック
- `use` プレフィックスを付ける
  - 例: `useAnimation`, `useTranslation`, `useFileHandler`

### ファイル名
- コンポーネント: パスカルケース（例: `TranslationList.tsx`）
- フック: キャメルケース（例: `useAnimation.tsx`）
- ユーティリティ: キャメルケース（例: `fileUtils.ts`）

## コードスタイル

### コンポーネント
- 関数コンポーネントと`React.FC`型を使用する
- JSXの複雑な式はコンポーネント外で変数に格納する
- 適切なコンポーネント分割を行い、1ファイルあたり300行を超えないようにする
- 複雑なレンダリングロジックは別の関数に抽出する

### フック
- カスタムフックに関連する状態とロジックをカプセル化する
- フック内の関数はuseCallbackでメモ化する
- 依存配列は必ず指定し、必要な依存関係のみを含める
- 副作用の適切な管理のためにuseEffectを使用する

### 型定義
- インターフェースや型エイリアスを使用して、Props型を明示的に定義する
- 汎用的な型は別ファイルに定義し、再利用する
- `any`型の使用を避け、具体的な型を指定する
- 必要に応じてユニオン型や交差型を活用する

```typescript
// 良い例
interface TranslationItem {
  key: string;
  value: string;
  isTranslated: boolean;
}

// 避けるべき例
const item: any = { /* ... */ };
```

### コメント
- 複雑なロジックには説明コメントを付ける
- 関数とカスタムフックには JSDoc コメントを使用する
- TODO コメントには担当者と期限を含める

```typescript
/**
 * 翻訳アイテムを検索する関数
 * @param items 翻訳アイテムの配列
 * @param query 検索クエリ
 * @param options 検索オプション
 * @returns フィルタリングされた翻訳アイテムの配列
 */
```

## パフォーマンス最適化

### メモ化
- 重い計算や頻繁に再レンダリングされる部分は `useMemo` を使用する
- イベントハンドラは `useCallback` でメモ化する
- 子コンポーネントは必要に応じて `React.memo` でラップする

### レンダリング最適化
- 大量のデータを表示する場合は仮想化ライブラリを検討する
- 状態更新は必要最小限にとどめる
- コンポーネントツリーの深さに注意し、必要に応じてコンテキストを使用する

## エラー処理
- try-catchブロックを使用して例外を適切に処理する
- エラー状態を明示的に管理し、ユーザーに適切なフィードバックを提供する
- デバッグ情報はコンソールに出力しない（本番環境では）

## テスト
- コンポーネントとフックにはユニットテストを書く
- 重要なユーザーフローには統合テストを書く
- テストカバレッジを意識する

## アクセシビリティ
- セマンティックなHTML要素を使用する
- ARIA属性を適切に設定する
- キーボードナビゲーションをサポートする

## ファイル構造
```
src/
├── components/      # 共通コンポーネント
│   ├── ui/          # 基本的なUI要素
│   └── features/    # 機能別コンポーネント
├── hooks/           # カスタムフック
├── utils/           # ユーティリティ関数
├── services/        # APIとの通信など
├── types/           # 型定義
├── contexts/        # Reactコンテキスト
└── pages/           # ページコンポーネント
```
